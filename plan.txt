# Kanban Dashboard — Projects Feature Plan
# Generated: 2026-03-19

## Goal
Add project-level kanban: user sees a Projects list first, picks a project, then sees that project's kanban board.

## New Project Fields
- name (required)
- description
- priority: low | medium | high
- status: planning | active | on_hold | completed
- planned_start_date, planned_end_date (DATE)
- actual_start_date, actual_end_date (DATE)
- created_by, created_at

## DB Changes (migrate_add_projects.py)
1. CREATE TABLE projects (id, name, description, priority, status, planned_start_date, planned_end_date, actual_start_date, actual_end_date, created_by FK→users, created_at)
2. ALTER TABLE tickets ADD COLUMN project_id INT UNSIGNED NULL FK→projects(ON DELETE SET NULL)

## Backend Changes (kanban-api/server.js)
- GET    /api/projects       — list all projects
- POST   /api/projects       — create project (auth)
- PUT    /api/projects/:id   — update project (auth)
- DELETE /api/projects/:id   — delete project (auth)
- GET    /api/tickets?project_id=X — filter tickets by project
- POST   /api/tickets — accept project_id field

## New UI Files
- kanban-ui/src/components/ProjectsPage.jsx  — grid of project cards, create/edit/delete
- kanban-ui/src/components/ProjectModal.jsx  — create/edit project form (all fields)

## Modified UI Files
- kanban-ui/src/App.jsx          — show ProjectsPage by default; pass selectedProject to KanbanBoard
- kanban-ui/src/components/KanbanBoard.jsx — accept project prop, filter tickets by project.id, show project name in header, Back button

## Navigation Flow
Login → ProjectsPage (list) → click project → KanbanBoard (project-scoped) → Back → ProjectsPage
