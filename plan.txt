### 3. Comments on Tickets

#### DB Migration (new file: migrate_add_comments.py)
CREATE TABLE IF NOT EXISTS comments (
  id          INT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY,
  ticket_id   INT UNSIGNED NOT NULL,
  user_id     INT UNSIGNED NOT NULL,
  body        TEXT NOT NULL,
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (ticket_id) REFERENCES tickets(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id)   REFERENCES users(id)
);

#### Backend (kanban-api/server.js) — add 3 routes:
- GET  /api/tickets/:id/comments  → list comments with author name (JOIN users)
- POST /api/tickets/:id/comments  → insert comment (auth required), return full row
- DELETE /api/comments/:id        → delete own comment only (check req.user.id === user_id)

#### Frontend (kanban-ui/src/components/TicketModal.jsx)
- When editing an existing ticket, fetch comments from GET /api/tickets/:id/comments
- Show scrollable list: author name, timestamp, body
- Add textarea + "Add Comment" button → POST /api/tickets/:id/comments
- Show delete button on own comments → DELETE /api/comments/:id

### 4. Deploy to Remote Server

  (outputs to kanban-api/public/)

Step B — Copy to server:
  rsync -avz kanban-api/ rajramani@192.168.1.150:/home/rajramani/kanban-api/

Step C — On server, install and start:
  cd /home/rajramani/kanban-api && npm install --omit=dev
  sudo cp /home/rajramani/kanban-api.service /etc/systemd/system/kanban-api.service
  # Edit service file to set real DB_PASSWORD and JWT_SECRET env vars

  sudo systemctl restart kanban-api

Step D — Run DB migrations on server:
  python3 setup_db.py                 # creates users + tickets if not exists
  python3 migrate_add_comments.py     # creates comments table
